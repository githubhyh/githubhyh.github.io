IDEA和DataGrip如何查看保存的数据库密码

发表于 2024-11-13 分类于 Idea 阅读次数：

通常我们在Idea或DataGrip中连接数据库后，不会另外保存数据库的用户名密码信息。但时间久了以后，当别人询问数据库密码时或我们需要别的工具进行连接时，我们不知道如何查看Idea或DataGrip中保存密码。目前官方是明确不支持内置查看密码的功能，且存储密码的文件是经过软件加密的，所以需要我们用官方提供的工具进行查看。

设置Master Password

必须要设置 Master Password，负责加密的密码文件无法打开，这是一个强制步骤

  • 打开 Database 窗口
  • 任意选择一个数据源
  • 点击 编辑 按钮
  • 点击 Save ，下拉选择 IDE Passwords settings
  • 点击 编辑 图标，选择 Set Master Password
  • 设置 Master Password，这个 password 就是用来解密密码文件的

数据库信息文件

数据库信息文件，记录了 url、username、uuid 以及一些其他的配置信息，我们需要 uuid 来唯一确认密码文件中的密码，做到一一对应

密码存储文件

点击 IDE Passwords Settings 按钮后，打开的页面中就有密码文件存储位置信息

在线查看

在线查看地址

  • 打开网页后，选择密码存储文件上传，之后输入 Master Password，就能看到存储的密码信息了，其中包含UUID信息，可以和我们本地的 dataSources.xml 文件中进行对应，能准确找到数据库密码

安装软件查看

软件下载地址

  • 下载软件并安装
  • 和在线查看过程一样，选择密码文件上传，然后输入 Master Password，就能正常看到密码信息了

说明

  • 解密出来的密码需要我们通过UUID来和数据库一一对应
